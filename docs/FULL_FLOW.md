## 一、环境概述

- 雷电模拟器 6 个实例多开（当前3个运行中）
- 分 2 组：A 组（实例 0/1/2）、B 组（实例 3/4/5）
- 每组 3 个机器人账号：1 个队长 + 2 个队员
- 每组还会有 **2 个真人玩家** 手动加入组队（共 5 人一队）
- 目标：两组同时匹配，匹配到同一对局后断网退出，循环

#### 弹窗关闭循环算法（实测优化版）

```
最大循环 = 20 次
连续大厅计数 = 0

while 循环 < 最大循环:
    截图

    # 1. 先检测是否已到大厅（最高优先级）
    if 匹配模板("开始游戏"红色按钮):
        连续大厅计数 += 1
        if 连续大厅计数 >= 3:
            return SUCCESS  # 弹窗清理完成
        wait(1.0)
        continue
    else:
        连续大厅计数 = 0

    # 2. 模板匹配 X 按钮（准备 3-4 种不同 X 样式）
    if 匹配模板(X按钮模板库):
        点击(X按钮位置)
        wait(0.8)
        continue

    # 3. OCR 扫描关键词并点击
    文字列表 = OCR(截图)
    if 包含("不再显示|不再弹出|不再提醒|今日内不再弹出"):
        先勾选该选项  # 减少未来弹窗数量
    if 包含("确定"):    点击("确定"位置)
    elif 包含("同意"):  点击("同意"位置)
    elif 包含("领取"):  点击("领取"位置)
    elif 包含("暂不需要"): 点击("暂不需要"位置)
    elif 包含("点击屏幕继续|点击继续"):
        点击(屏幕中央)
    else:
        # 4. 兜底：点击屏幕中央
        点击(屏幕中央)

    wait(1.0)

return TIMEOUT  # 弹窗过多，告警
```

**关键优化点**：
- 步骤7弹窗中的"今日内不再弹出"（7.1.1截图底部）、"两天内不再提醒"（zhoaduiyou截图），**优先勾选**可减少后续启动的弹窗数量
- 赛季结算连续链（B1-B4）无需识别具体内容，快速连点即可
- 最大循环20次足够覆盖7个弹窗+偶发额外弹窗

## 五、关键识别策略

### 弹窗通用关闭策略

不管什么弹窗，它们有共性特征：
- 有一个半透明暗色遮罩层
- 弹窗本体是白色/浅色的矩形区域
- 右上角或右侧有一个 X 关闭按钮

**识别方式**：找画面中最大的浅色矩形 → 在矩形的右上角区域找深色/对比色的 X 形状或圆形按钮 → 点击。

### 组队人数检测

优先使用 OCR 读取队伍人数文字（如 "4/5"）。
备选：检测空位占位图标。
不推荐：靠游戏人物模型判断（动态、不稳定）。

### 对手校验

**第一道过滤**：两组进入加载的时间差 < 3 秒 → 可能匹配到一起了。
**第二道校验**：OCR 读对手队伍第一个玩家名字 → 是否等于另一组某成员名字。
两道都通过才认定匹配成功。

### 匹配状态判定

- "匹配中"状态：画面有计时数字在变化（1 2 3 4 5...）
- "匹配成功"状态：画面突然切换到加载地图页
- "匹配超时"：计时超过 60 秒

### 结论：通过修改文件拦截弹窗 — ❌ 不可行
- 弹窗状态存储在 GVAS 二进制格式（UE4 序列化），无法简单文本修改
- itop.xml 登录信息经过 MSDK 加密
- popup/html 内容也是加密的
- **最佳策略**：在脚本中勾选"今日不再弹出"/"不再提醒"选项，从UI层面减少弹窗
