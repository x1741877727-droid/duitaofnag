## 数据档案实时性 — 已确认是实时

WebSocket 链路:
```
Decision.finalize() → recorder._notify_listeners()
  → api_live._on_decision_finalized()
  → /ws/live broadcast
  → useLiveStream 收到 'decision' event
  → store.pushLiveDecision()
  → ArchiveView useMemo(liveCount) 触发
  → fetchDecisionList(sessionId) 重拉 (LIVE session only)
```

**已知缺陷** (不修, 可接受):
- 每条决策触发**全量重拉**, 不是增量 push (一秒 5 条 → 5 次 GET)
- 决策**详情面板**不自动刷新 (需重选 decisionId)
- 历史 session (非 isCurrent) 不跟新决策 — 合理
